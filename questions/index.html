Question:

Hey
We currently use swiftype. I'd like to learn what the main differences with respect to that.
Thanks. Hugo.

Answer:

Hi Hugo,
Thanks for your interest in Algolia! While we share some similarities with Swiftype, one of the main differences that sets us apart from them is that instead of using a web crawler based on older, existing technology to build an online index of your site (which is a large and complex task), Algolia builds an index of your site from data pushed to us directly from you (either through our online dashboard or our REST API). Algolia's engine was built from the ground-up by our engineers, giving us full control and flexibility of indexing and search. By using our method, the search task is much simpler and provides higher performance and more intelligent advantages of a full in-house search engine operating on your back-end database. This allows Algolia’s search response speed to to be significantly faster since it’s searching a single site rather than the entire web. Algolia searches are also smarter and more site specific than a generalized web text search.

Another feature that sets Algolia apart is our dedication to speed by optimizing for every millisecond, from how the engine works to our infrastructure. We are the only search provider to propose a Distributed Search Network (DSN) with 14 worldwide locations. DSN enables you to automatically duplicate your search engine and your data in multiple locations around the world (this is a complete copy of the search engine with all data, not just a cache). This way, when one of your users searches for something, the query will be sent to the closest datacenter, which will greatly reduce the latency.

These differences are among the many of our rich feature set. If you have any more specific questions please let me know, I would be happy to answer them or set up a call to discuss more about using our product.

Thanks, Justin

Hi, thank you for your interest in Algolia! Please see my responses to your questions below highlighted in red.

1. We have three domains right now that contain different data. Would we be able to use a single Algolia account with different endpoints for each of those domains? For example, if we have less than 1M documents across the three domains, can we use a single Growth account?
[JM] - Yes, you would be able to use your three domains under a single Growth account. In fact, there is no limitation to the amount of domains you can have.

2. We have a staging environment that also has three domains. The size of our data in the staging environment is fairly close to production to test performance in a realistic way. Queries per second is obviously much, much lower. What is the pricing for a non-production environment?
[JM] - Pricing is the same for both production and non-production environments. In order to provide the same performance on features on non-production environments, we need to use the same hardware setup and so a discount is not possible for this.

3. Does filtering on facets restrict the documents returned or the facets returned?
[JM] - Filtering on facets will restrict on documents.

4. If the answer to 3 is documents, can we filter on facets using ranges? Or alternatively, can we provide arrays for faceted attributes where we can return the document if any of the values in the array match the filter (similar to querying against array fields in MongoDB)? As an example scenario, we have categories containing sub-categories which contain products. When in the parent category, we need to be able to search products in all of its sub-categories. In the sub-categories, we need to be able to search products in only that sub-category. Let me know if anything is unclear. It looks like a great product so I'm excited to hear back!
[JM] -You cannot filter over facets using ranges; however, you can filter over numeric and date attributes with math operations such as “<“,”>”,”=“,”<=“,”>=“. And yes, arrays are accepted for faceted attributes. Regarding your example scenario, you can achieve your desired results by using the algoliasearch-helper-js library to specify a rootPath from where you would like to start filtering hierarchical values. The link to the library github page provided will give more explanation on using these faceting features and show code examples of how to use them.
